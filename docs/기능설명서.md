# Content Hub — 기능 설명서

> Higgsfield 로 만든 영상·이미지를 팀이 **한곳에 모아 보고·정리·검색·공유·재사용·계보추적** 하는 도구.
> 이 문서는 **기능별 상세 매뉴얼**이다. 빠른 시작만 필요하면 [사용설명서.md](사용설명서.md),
> 서버 운영 상세는 [SERVER.md](SERVER.md), 코드 구조는 [ARCHITECTURE.md](ARCHITECTURE.md) 를 본다.
>
> 최종 갱신: 2026-06-19 · 기준 포트 **8010** · 로그인 기본 **ON**

---

## 0. 핵심 개념 한 장

- **서버 한 대 + 브라우저 접속**: 팀원은 설치 없이 브라우저로 주소만 친다.
- **서버는 생성하지 않는다**: 생성·재생성은 **각자 본인 PC 의 Higgsfield CLI**(본인 크레딧). 서버는 결과를 모아 보여주는 **공유 DB**.
- **토큰은 내 PC 밖으로 안 나간다**: 서버는 Higgsfield 자격증명을 저장하지 않는다.
- **두 가지 로그인 구분**:
  - **허브 로그인**(브라우저 계정) = 신원·권한·"내 작업" 분리.
  - **Higgsfield CLI 인증**(각 PC) = 실제 생성 주체.

## 1. 역할별 시작하기

### 1-A. 서버 운영자 (서버 PC 1대)

```
MV_server.bat
```

- 프론트 빌드 → 백엔드 기동까지 한 번에. 최초 1회는 의존성 설치로 수 분.
- 기본 **포트 8010**, **로그인 ON**(`CONTENT_HUB_AUTH=1`).
- 접속:
  - 같은 PC: `http://127.0.0.1:8010` (★ `localhost` 는 IPv6 폴백으로 ~200ms 느림 → `127.0.0.1` 권장)
  - 같은 네트워크(팀원): `http://<서버 PC IP>:8010` (IP 는 서버에서 `ipconfig` → IPv4)
- 사전 요건: Python(`pip install -r backend/requirements.txt`), Node.js.
- 로그인 끄기(개인/개발, 모두 "나"): bat 의 `CONTENT_HUB_AUTH` 를 `0` 으로.

### 1-B. 팀원 (각자 PC — 생성/푸시 에이전트)

서버에 결과를 올리거나 허브의 생성 요청을 본인 CLI 로 실행하려면 본인 PC 에서 에이전트를 켠다.
**추가 설치 불필요**(표준 라이브러리만):

```
python agent_push.py --server http://<서버IP>:8010 --email <내이메일> --watch 30
```

- `--watch 30`: 30초마다 ① 허브의 내 생성요청을 내 CLI 로 실행 + ② 내 새 결과물을 서버로 push.
- `--token <세션토큰>`: 로그인 입력 없이(자동화/테스트용).
- 전제: 본인 PC 에 `higgsfield auth login` 으로 CLI 가 로그인돼 있어야 한다.
- 에이전트를 안 켜도 **탐색·정리·공유는 가능**(생성만 "로컬 대기" 카드로 남았다가 켜면 실행됨).

### 1-C. 일반 사용자 (브라우저만)

주소 접속 → (로그인 모드면) 회원가입/로그인 → 라이브러리 사용.

```
┌──────────────────────────────────────────────────────────┐
│ ⬡ Content Hub  [워크스페이스]  검색  👤이름  내작업 팀작업 캔버스 │ ← 상단바
├────────────┬─────────────────────────────────────────────┤
│ 프로젝트    │                                             │
│ 컬러        │            결과물 썸네일 그리드               │ ← 라이브러리
│ 자동태그    │                                             │
│ 작업자      │                                             │
│ 공유        │   ┌─────────────────────────────────────┐  │
│            │   │ 프롬프트 입력 (하단 도크, Ctrl/⌘+K)    │  │
└────────────┴───┴─────────────────────────────────────┴──┘
```

- **상단바**: 로고(⬡, 클릭 시 관리자 창)·워크스페이스·검색·내 이름(계정 메뉴)·탭.

- **가운데 그리드**: 무한 스크롤. 카드 클릭=크게 보기, 휠(중간)클릭=정보 팝업.
- **하단 도크**: 프롬프트 입력 → 생성.

## 3. 라이브러리 (탐색·정리)

### 보기
- **무한 스크롤**(키셋 기반) + 가상 스크롤로 대량도 가볍게. **그리드/리스트** 토글, **날짜 그룹**.
- **카드**: 썸네일(영상은 호버 시 재생), 상태 pill(로컬 대기/로컬 생성중/완료/실패), 컬러 마커, 타입 배지, 작성자.

### 분류 (선택 후 단축키)
| 키 | 동작 |
|---|---|
| `r` / `g` / `b` | 컬러 마커 빨강/초록/파랑 |
| `s` | 소스로 등록(@소스명) |
| `#` | 태그 입력 |
| 카드 휠클릭 | 정보 팝업(프롬프트·소스·크레딧·소속 프로젝트·프로젝트 변경) |

### 벌크(여러 개 한꺼번에)
- 선택: 카드 클릭·**Shift/Ctrl**·**Ctrl+A**·**마퀴 드래그**·날짜 그룹 헤더 선택.
- 일괄: 삭제 / 복원 / 영구삭제 / 공유 / 프로젝트 귀속.

### 검색
- 상단 검색창에 **프롬프트·태그 부분일치**. 3자 이상은 FTS5(빠름), 미만은 LIKE 폴백.

### 필터(사이드바)
- **컬러·자동태그·작업자·공유여부·프로젝트**. 여러 필터는 AND 로 합성.

## 4. 생성 / 재생성

- **생성**: 하단 도크에 프롬프트 입력 + 모델/비율 선택 → 생성. `Ctrl/⌘+K` 로 도크 포커스.
  - `@` 입력 → 소스(레퍼런스) 피커, `#` → 태그/프롬프트 조각 피커.
- **재생성**: 카드 오버레이의 ↻ — 같은 프롬프트·레퍼런스로 다시. 원본과 `derived` 계보가 남고 같은 프로젝트로 자동 귀속.
- **동작 방식**(중요): 버튼은 **서버에 요청만** 남긴다 → 카드가 즉시 **"로컬 대기"** 로 뜨고 → 내 PC 에이전트(`--watch`)가 가져가 **내 CLI 로 실행** → 완료되면 카드가 채워진다.
  - 에이전트가 꺼져 있으면 "로컬 대기" 로 남았다가 켜는 즉시 실행. 결과·크레딧은 실행한 사람 것.
  - ⚠️ **생성은 유료**(크레딧 소모). 텍스트·URL 레퍼런스는 OK, 로컬파일/`asset:` 레퍼런스는 타 PC 에이전트가 아직 못 푸는 한계.

## 5. 프로젝트 (작업 묶음)

여러 작업이 섞이지 않게 결과물을 묶는 **팀 공통 분류**(개인 컬러·태그와 별개).

| 하고 싶은 것 | 방법 |
|---|---|
| 만들기 | 사이드바 **프로젝트** 섹션의 `+` |
| 그 프로젝트만 보기 | 이름 클릭(다시 클릭=해제), **미분류** 항목도 있음 |
| 결과물 담기 | 카드 선택 → 하단 **📁 프로젝트에 담기** |
| 생성과 동시에 담기 | **특정 프로젝트로 필터한 상태에서 생성** → 자동 귀속 |
| 한 개만 옮기기 | 카드 휠클릭 → 정보 팝업의 프로젝트 드롭다운 |
| 이름변경·보관·삭제 | 사이드바 항목 호버 ✎/✕, 또는 관리자 창 |

> 프로젝트를 삭제해도 **결과물은 안 지워지고 '미분류'로** 돌아간다. 보관(archived)은 콜드 분리.

## 6. 팀 작업 · 등급(골드)

- **공유(발행)**: 카드 오버레이의 **S** — 단일클릭=공유/해제, 더블클릭=**최종(골드)** 지정.
  - 최종(골드)은 그 프로젝트의 supervisor/PM 권한이 있어야 지정 가능. 골드는 공유 잠금.
- **팀 작업 탭**: 공유된 결과물 갤러리. 가져오면 로컬 복제 + 계보 기록.
- **번들 내보내기/가져오기**(상단바): 결과물을 JSON 번들 파일로 주고받기 — 프롬프트·소스·태그·코멘트까지 보존(로그인 없이도 가능).
- **동기화**: 서버 PC 에 한해, Higgsfield 잡(다른 기기·웹 포함)을 주기적으로 흡수(과도기 편의).

## 7. 캔버스 탭 — 씬 캔버스 · 히스토리 보기(계보)

**캔버스 탭** 안에 두 가지 뷰가 있다: 자유 배치용 **씬 캔버스**와 계보 트리용 **히스토리 보기**
(상단 씬 바에서 전환·씬 추가).

- **씬 캔버스**: 빈 캔버스에 결과물 카드를 자유 배치하고 카드끼리 **연결선·태그**를 편집하는
  작업판. 여러 개의 씬을 만들 수 있다. 저장은 **이 브라우저 로컬**(localStorage, 서버 무관).
- **히스토리 보기** = 원본 → 파생을 가로 트리로 그리는 **계보 보드**(`HistoryBoard`).
  - 엣지 종류: **실선 `derived`**(재생성·가져오기, 강한 1부모) · **점선 `reference`**(@소스로 만듦, 약한 다부모).
  - 무한 캔버스(휠 줌·미들클릭 팬), 마퀴 선택·비교·정보·다운로드·재생성·드래그 이동.
  - 단축키: **`d`** 비활성화(회색), **`l`** 자동 정렬. 최종(골드) 강조.
- **가계 패널**(`HistoryPanel`) + 미니 트리로 한 카드의 연결된 가계를 빠르게 본다.

## 8. 코멘트 & 알림

- 공유된 결과물에 **코멘트 스레드**(답글 지원). 카드의 **C 뱃지** = 내가 아직 확인 안 한 알림.
- **알림이 울리는 경우는 둘뿐**: ① **내가 만든 생성물**에 달린 코멘트, 또는 ② **내 코멘트에 달린 답글**.
  - 그 외 코멘트는 패널에서 볼 수 있지만 내 C 뱃지를 켜지 않는다.
- **확인 방식**: 카드 C 클릭 → 패널에서 **NEW** 표시된 코멘트를 클릭(또는 답글)하면 그것만 확인 처리. 모두 확인하면 C 뱃지가 꺼진다(코멘트 단위 seen).
- 새 글/삭제는 실시간(WebSocket)으로 즉시 반영.

## 10. 계정 · 등급 · 관리자 창

좌측 상단 **⬡ 로고 클릭 → 관리자 창**.

- **멤버 · 등급**: 작성자 목록 + 등급(C0~C5). C0=관리자(전부), C1=프로젝트 관리자, C2~C5=멤버. 드롭다운으로 변경.
  - (내부 역할: admin / product_manager / product_director / production_director / member 전역 역할 + 프로젝트 역할 project_manager / supervisor / editor)
- **프로젝트**: 전체 프로젝트 생성·이름변경·보관·삭제.
- **로그인 계정**(로그인 모드일 때): 가입 대기 계정 **승인/거부**, 등급 부여.

### 로그인 보안 모드
- 기본 ON(`CONTENT_HUB_AUTH=1`). **첫 가입자 = 관리자**, 이후 가입자는 **승인 대기 → 관리자 승인** 후 로그인.
- 로그인 안 하면 결과물·미디어·실시간 갱신 전부 차단. 우측 상단 계정 칩으로 로그아웃.
- 비밀번호는 해시 저장, 세션은 위조 불가 서명 토큰. 끄려면 `CONTENT_HUB_AUTH=0`(데이터는 그대로).

## 13. 운영 · 백업

- 모든 데이터는 서버 PC `backend/data/`(DB·미디어·공유). 미디어는 `data/media/<sha[:2]>/...` 샤딩.
- **자동 백업**: 하루 1회 + 시작 시, 최근 7개 보관. 수동은 `POST /api/backup`.
- 다른 디스크/NAS 권장: `set CONTENT_HUB_BACKUP_DIR=\\NAS\backup`.
- **변경 반영**: 백엔드 코드 변경 → **서버 재시작 필수**(`--reload` 금지). 프론트 변경 → `npm run build` + 브라우저 **Ctrl+F5**.
- **DB 초기화/점검**: `cd backend && python -m app.db init`(멱등).
- PostgreSQL 전환은 `deploy/POSTGRES.md`, 리버스 프록시는 `deploy/`(nginx/Caddy).

## 14. 자주 묻는 것

**Q. 팀원이 설치해야 하나요?**
브라우저 접속만으로 탐색·정리·공유 OK. 단 **본인 결과를 올리거나 허브에서 생성을 돌리려면** 본인 PC 에서 `agent_push.py --watch` 를 켜야 한다(추가 설치 없음).

**Q. 생성은 누구 계정으로 되나요?**
각자 본인 PC 의 Higgsfield CLI 계정으로. 서버는 토큰을 갖지 않고 결과만 모은다.

**Q. 허브엔 oz1 으로 로그인했는데 동기화는 jay 로 잡혀요.**
허브 로그인과 그 PC 의 CLI 계정은 별개다(§0). 그래서 각자 본인 PC 에서 본인 CLI + 에이전트로 push 하는 모델이다.

**Q. 화면/갱신이 안 떠요.**
서버가 켜져 있는지, 방화벽에서 포트 **8010** 이 열렸는지 확인. 같은 PC 는 `127.0.0.1:8010` 권장.

**Q. 실서버로 옮기려면?**
이 폴더째 복사 → `MV_server.bat`(또는 동등 명령). 상대경로 기반이라 무변경 작동. 상세는 [SERVER.md](SERVER.md).
