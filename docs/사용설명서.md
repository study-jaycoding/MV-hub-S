# Content Hub 사용 설명서

Higgsfield CLI로 만든 영상·이미지 생성물을 **한 곳에 모아 보고, 정리하고, 팀과 공유**하는
로컬 우선 도구입니다. 이 폴더(`content-hub-server`)는 **서버 배포용**으로, 한 대(서버 PC)에서
돌리면 팀원들이 브라우저로 접속해 함께 씁니다.

---

## 1. 빠른 시작

서버 PC에서:

```
MV_server.bat
```

이 한 줄이 프론트엔드를 빌드하고 백엔드를 띄웁니다. 처음 한 번은 의존성 설치로 몇 분 걸립니다.

접속:
- **같은 PC**: http://localhost:8010
- **같은 네트워크(팀원)**: `http://<서버 PC의 IP>:8010`
  - IP는 서버 PC에서 `ipconfig` → IPv4 주소로 확인

> 필요 조건: Python(`pip install -r backend/requirements.txt`), Node.js, 그리고 본인 PC에
> 로그인된 Higgsfield CLI(생성·동기화는 본인 계정으로 동작).

---

## 2. 기본 화면

```
┌──────────────────────────────────────────────────┐
│ ⬡ Content Hub  [워크스페이스] 👤이름  내작업 팀작업 캔버스 │  ← 상단바
├────────────┬─────────────────────────────────────┤
│ 프로젝트    │                                      │
│ 컬러        │        결과물 썸네일 그리드           │  ← 라이브러리
│ 자동태그    │                                      │
│ 작업자      │                                      │
│ 공유        │  ┌────────────────────────────────┐ │
│            │  │ 프롬프트 입력 (하단 도크)        │ │
└────────────┴──┴────────────────────────────────┴─┘
```

- **상단바**: 좌측 로고(⬡), 워크스페이스, 내 이름, 탭(내 작업 / 팀 작업 / 캔버스)
  - **내 작업**: 내가 만든 결과물 라이브러리 · **팀 작업**: 팀이 공유한 것
  - **캔버스**: 결과물을 자유 배치하는 **씬 캔버스**(카드·연결·태그, 이 브라우저에 로컬 저장)
    와, 원본→파생을 트리로 보는 **히스토리 보기**가 함께 있는 탭
- **좌측 사이드바**: 프로젝트·컬러·자동태그·작업자·공유 필터
- **가운데**: 결과물 그리드 (휠클릭 = 정보 팝업, 클릭 = 크게 보기)
- **하단 도크**: 프롬프트 입력 → 생성

### 자주 쓰는 동작
- **생성**: 하단 프롬프트에 입력 + 모델 선택 → 생성. `Ctrl/⌘+K`로 프롬프트 포커스
- **다중 선택**: 카드 클릭으로 선택(여러 개) → 하단에 선택 바 등장
- **컬러/태그/소스 분류**: 선택 후 단축키 `r`·`g`·`b`(컬러), `s`(소스), `#`(태그)
- **정보 보기**: 카드 휠(중간) 클릭 → 프롬프트·소스·크레딧·**소속 프로젝트** 팝업

## 3. 프로젝트 (작업 묶음)

여러 작업이 섞이지 않게 결과물을 **프로젝트 단위로 묶는** 기능입니다. (개인 태그·컬러와 다른,
팀 공통의 분류입니다.)

| 하고 싶은 것 | 방법 |
|---|---|
| 프로젝트 만들기 | 좌측 사이드바 **프로젝트** 섹션의 `+` |
| 그 프로젝트만 보기 | 사이드바에서 프로젝트 이름 클릭 (다시 클릭 = 해제) |
| 미분류만 보기 | **미분류** 항목 클릭 |
| 결과물 담기 | 카드 여러 개 선택 → 하단 **📁 프로젝트에 담기** → 프로젝트 선택 |
| 생성과 동시에 담기 | **특정 프로젝트로 필터한 상태에서 생성** → 자동으로 그 프로젝트에 들어감 |
| 한 개만 옮기기 | 카드 휠클릭 → 정보 팝업의 **프로젝트** 드롭다운 |
| 이름변경·삭제 | 사이드바 프로젝트에 마우스 올리면 ✎(이름)·✕(삭제) |

> 프로젝트를 삭제해도 **결과물은 지워지지 않고 '미분류'로 돌아갑니다.**
> 재생성한 결과물은 원본과 같은 프로젝트에 자동으로 들어갑니다.

---

## 4. 멤버 등급 · 관리자 창

좌측 상단 **"⬡ Content Hub" 로고를 클릭**하면 관리자 창이 열립니다.

- **멤버 · 등급**: 생성물 작성자(멤버) 목록과 등급(C0~C5)
  - C0 = 관리자(전부) / C1 = 프로젝트 관리자 / C2~C5 = 멤버
  - 등급은 드롭다운으로 변경
- **프로젝트**: 모든 프로젝트를 생성·이름변경·보관·삭제
- **로그인 계정**(아래 보안 모드일 때만): 가입 대기 계정 **승인 / 거부**, 등급 부여

---

## 5. 팀 서버 — 로그인 보안 (선택)

기본은 **누구나 접속 가능**(개인 PC·사내망). 접근을 제한하려면 **로그인 모드**를 켭니다.

서버 실행 시:
```
set CONTENT_HUB_AUTH=1
MV_server.bat
```

켜면:
1. **첫 가입자 = 관리자**가 됩니다(서버를 처음 띄운 사람이 회원가입).
2. 이후 가입자는 **승인 대기** 상태 → 관리자가 관리자 창에서 **승인**해야 로그인 가능.
3. 로그인하지 않으면 아무것도 못 봅니다(결과물·미디어·실시간 갱신 전부 차단).
4. 우측 상단 계정 칩을 클릭하면 로그아웃.

> 비밀번호는 안전하게 해시 저장되고, 세션은 위조 불가한 서명 토큰을 씁니다.
> 끄려면 `CONTENT_HUB_AUTH`를 빼고(또는 0) 다시 실행 — 데이터는 그대로입니다.

---

## 6. 팀 작업 (로그인 없이도)

- **내보내기 / 가져오기**(상단바): 결과물을 번들 파일로 주고받기 — 프롬프트·소스·태그·코멘트까지 보존
- **동기화**: Higgsfield에서 만든 잡(다른 기기·웹 포함)을 주기적으로 자동으로 끌어옵니다
- **코멘트**: 공유된 결과물에 누구나 의견을 남길 수 있습니다(C 뱃지 = 안 읽은 코멘트)

---

## 7. 데이터 · 백업

- 모든 데이터는 서버 PC `backend/data/`에 있습니다(DB·미디어·공유).
- **DB는 자동 백업**됩니다(하루 1회 + 시작 시, 최근 7개 보관). 수동 백업은 관리자가
  `POST /api/backup`으로도 가능.
- 백업은 가능하면 다른 디스크/NAS로: `set CONTENT_HUB_BACKUP_DIR=\\NAS\backup`

---

## 8. 자주 묻는 것

**Q. 팀원이 자기 PC에 설치해야 하나요?**
아니요. 서버 한 대만 돌리고, 팀원은 **브라우저로 주소만** 접속합니다.

**Q. 생성은 누구 계정으로 되나요?**
각자 본인 PC의 Higgsfield CLI 계정으로요. 서버는 결과를 모아 보여줄 뿐, 토큰을 갖지 않습니다.

**Q. 화면이 안 떠요 / 갱신이 안 돼요.**
서버가 켜져 있는지, 방화벽에서 포트(기본 8010)가 열렸는지 확인하세요. 백엔드는 항상 켜둡니다.

**Q. 실서버로 옮기려면?**
이 폴더를 그대로 서버에 복사 → `MV_server.bat`(또는 동등 명령). 상대경로 기반이라 무변경
작동합니다. 자세한 건 `SERVER.md` 참고.
